Leçon 2 · Claude Code × MZD

Écrire ton premier skill MZD

Anatomie d'un SKILL.md — et construction d'un skill qui fetche les tâches Notion et produit un brief de coordination.

Rappel de la leçon 1

Un skill Claude Code ne doit jamais raisonner depuis sa connaissance paramétrique sur des données organisationnelles. Il doit d'abord fetcher le contexte réel, puis raisonner. Le pattern est : 

FETCH → REASON → ACT

Cette leçon te fait écrire ce pattern dans un vrai fichier SKILL.md.

Anatomie d'un SKILL.md

Un skill est un fichier Markdown avec deux parties : un frontmatter YAML, et un corps qui décrit ce que l'agent doit faire. 

---
name: nom-du-skill
description: "Décrit quand et pourquoi l'agent doit déclencher ce skill."
disable-model-invocation: true   # ← si tu veux l'invoquer toi-même
---

## Étape 1 — Fetch le contexte
[Ce que l'agent doit lire depuis les vrais outils]

## Étape 2 — Raisonner
[Ce que l'agent doit faire avec ce contexte]

## Étape 3 — Agir
[L'output ou l'action finale]
Trois champs frontmatter qui comptent name — identifiant du skill (utilisé dans /nom-du-skill)
description — présent → l'agent peut l'invoquer seul ; absent → tu l'invoques toi-même (user-invoked)
disable-model-invocation: true — force le mode user-invoked

Deux règles pour des étapes robustes

Règle 1 — Chaque étape se termine sur un critère de complétion

Un completion criterion est la condition qui dit à l'agent "tu peux passer à la suite." Sans lui, l'agent peut sauter des étapes avant d'avoir fini — c'est ce que Matt Pocock appelle la premature completion.

❌ Vague — risque de saut
## Étape 1 — Lire Notion

Lire les tâches Notion de la MZD.
✅ Net — critère explicite
## Étape 1 — Fetch les tâches Notion

Appelle notion-query-database-view sur la base "Tâches MZD". Continue quand toutes les tâches de la semaine en cours sont dans le contexte.

Règle 2 — Ne montre que les post-completion steps nécessaires

Plus l'agent voit d'étapes à venir, plus il est tenté de les bâcler pour "avancer". Pour les skills courts (2-3 étapes), ce n'est pas un problème. Pour les skills longs, Matt Pocock recommande de séparer les étapes en plusieurs fichiers.

Construire le skill brief-hebdo

Ce skill fetche les tâches Notion et les messages Slack de la semaine, puis produit un brief de coordination pour l'équipe MZD. 

---
name: brief-hebdo
description: >
  Produit le brief de coordination hebdomadaire de la MZD.
  Utilise quand l'équipe veut un résumé actionnable des tâches
  Notion et des échanges Slack de la semaine.
---

## Étape 1 — Fetch les tâches Notion de la semaine

Appelle `notion-query-database-view` sur la base de données
"Tâches MZD". Filtre par : date d'échéance dans les 7 prochains jours,
ou statut "En cours". Continue quand chaque tâche avec son statut,
son assigné et sa date est dans le contexte.

## Étape 2 — Fetch les messages Slack récents

Appelle `slack_read_channel` sur le canal principal de la MZD
pour les 3 derniers jours. Continue quand les fils importants
(décisions, blocages, demandes) sont identifiés dans le contexte.

## Étape 3 — Synthétiser le brief

Sur la base du contexte fetchté (uniquement ce contexte — ne pas
inventer de tâches ou de décisions), rédige un brief structuré :

- **Cette semaine** : 3 priorités maximum, avec la personne responsable
- **Blocages** : ce qui nécessite une décision ou une aide
- **À noter** : informations pertinentes des échanges Slack

Le brief doit être lisible en moins de 2 minutes.

## Critère de complétion

Le brief est complet quand chaque tâche "En cours" ou à échéance
cette semaine est mentionnée, et quand les blocages sont explicites.
Remarque le "uniquement ce contexte" C'est le garde-fou anti-hallucination : on interdit explicitement à Claude d'inventer des tâches ou des décisions qui ne seraient pas dans le contexte fetchté. C'est la traduction directe du principe de la leçon 1.

Où placer ce fichier

Les skills Claude Code vivent dans le dossier .agents/skills/ du repo. Pour le skill brief-hebdo : 

.agents/
└── skills/
    └── brief-hebdo/
        └── SKILL.md   ← ton fichier

Une fois en place, tu l'invoques depuis Claude Code avec : 

/brief-hebdo

Ce que tu peux faire maintenant

Dis à Claude "crée le skill brief-hebdo" dans ce repo. Il va générer le fichier .agents/skills/brief-hebdo/SKILL.md avec le contenu vu dans cette leçon, et tu pourras l'invoquer directement.

Si tu veux d'abord explorer les MCP tools disponibles pour Notion et Slack, demande : "liste-moi les tools Notion disponibles dans cette session."
