# Résumés de CR CA

**Leçon 16** — Skill `resume-cr-ca` : transformer un compte-rendu de CA (document formel, public juridique) en résumé lisible pour les bénévoles.

## Insights clés

**Pattern transformation.** Troisième pattern après triage (brief-hebdo) et orient (gouvernance-partager). Ce qui le distingue : la source et l'output ont des publics cibles différents. Le CR CA est écrit pour le CA. Le résumé est écrit pour les bénévoles. Ce changement de public crée des tensions spécifiques que les deux autres patterns n'ont pas.

**Traduis le langage, préserve les faits.** Simplification du vocabulaire autorisée. Inférence d'intentions ou de conclusions non présentes dans le CR : interdite. Tout ce qui n'est pas dans le CR source ne peut pas entrer dans le résumé. Cette règle est plus stricte que pour les autres patterns parce que la source a une valeur légale — elle engage le CA.

**Filtre déclaré, pas silencieux.** Les points administratifs (quorum, signatures, approbation du PV précédent) n'intéressent pas les bénévoles — mais les omettre sans le dire est trompeur. Le pied de page mentionne explicitement les points non développés avec un lien vers le CR complet. Le bénévole sait ce qu'il n'a pas lu.

**Structure en trois blocs.** Décisions (une phrase par décision, langage courant) + contexte clé optionnel (seulement si explicitement dans le CR) + pied de page (date, lien source, points omis). Le bloc contexte disparaît quand rien d'utile n'est mentionné dans le CR — il ne s'invente pas.

## Implications

Les trois patterns sont maintenant en place : triage, orient, transformation. Ils couvrent ensemble les quatre objectifs de la mission MZD. La prochaine étape : exécuter ces skills sur de vraies données MZD (CR réel, vraie page Notion de gouvernance) pour valider qu'ils fonctionnent sur le terrain, pas seulement en conception.
